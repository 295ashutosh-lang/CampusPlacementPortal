# Project Timeline

The development of the **Campus Placement Portal** is planned over a period of **14 weeks** following the Agile Software Development Model. The project is divided into multiple phases, where each phase focuses on completing a specific set of tasks before moving to the next stage.

| Week       | Phase                       | Activities                                                                                               | Deliverables                                    |
| ---------- | --------------------------- | -------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| Week 1     | Requirement Analysis        | Gather requirements, identify stakeholders, define project scope, prepare SRS.                           | Software Requirement Specification (SRS)        |
| Week 2     | Project Planning            | Prepare project schedule, choose technology stack, identify risks, allocate tasks.                       | Project Plan & Timeline                         |
| Week 3     | System Design               | Design database schema, system architecture, REST APIs and UI wireframes.                                | ER Diagram, Architecture Diagram, UI Wireframes |
| Week 4-5   | Frontend Development        | Develop React components, routing, authentication pages and dashboard UI.                                | Functional Frontend                             |
| Week 6-8   | Backend Development         | Develop Spring Boot services, REST APIs, JWT authentication and MongoDB integration.                     | Backend APIs & Database                         |
| Week 9     | Integration                 | Connect frontend with backend APIs and perform data integration.                                         | Integrated Web Application                      |
| Week 10    | Testing                     | Perform unit testing, integration testing, API testing and fix defects.                                  | Tested Application                              |
| Week 11    | Deployment                  | Deploy frontend, backend and database. Configure CI/CD pipeline.                                         | Live Application                                |
| Week 12-14 | Documentation & Maintenance | Prepare project documentation, improve performance, fix remaining bugs and implement minor enhancements. | Final Report & Stable Application               |
